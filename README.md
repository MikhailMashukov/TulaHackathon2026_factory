# Весёлый плановик — MVP

Веб-симулятор планирования производства для кейса task3 (Райтек ДТГ) на TulaHack 2026.

## Цель проекта и соответствие кейсу task3

- Показать в браузере «живой» упрощённый APS-цикл из условия: план → событие → перепланирование → оценка результата.
- Выполнить ключевые требования кейса: 5+ станков, 3+ товара с разными маршрутами, диаграмма Ганта, случайные события, перепланирование, баллы и пояснения по действиям.
- Сфокусироваться на демонстрации кооперации участков (а не на тяжёлом MILP/CP-SAT) и играбельности.

## Товары и технологические цепочки

- **fan (вентилятор)**: штампуем из пластика лопасти и корпус (ПрессПласт), обрезаем, шлифуем их (Доводка), собираем (Сборка), проверяем и упаковываем (ОТКУпак).
- **lamp (лампа)**: штампуем корпус (ПрессПласт), обрезаем, шлифуем (Доводка), собираем (Сборка), проверяем и упаковываем (ОТКУпак).
- **plastic_cap (пластиковая крышка)**: штампуем (ПрессПласт), проверяем и упаковываем (ОТКУпак).

Все детали рецептов находятся в `src/data/sample_orders.json` и могут быть изменены.

## Почему игра должна быть интересной

Данные моменты в целом продуманы и описаны в docs/factory_gameplay.md, но не реализованы.

- В игре есть конфликт локальной и глобальной оптимальности: «ускорить один станок» часто ухудшает общий срок из-за узких мест.
- Случайные события ломают «идеальный» план и вынуждают быстро балансировать между дедлайнами, загрузкой и качеством.
- Игрок получает мгновенную обратную связь: Гант, карточки станков, события и score показывают цену каждого решения.
- Время не останавливается даже когда вы смотрите на план и меняете его.

## Запуск

```bash
pip install -r requirements.txt
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
# открыть http://localhost:8000/
```

Тесты:
```bash
pytest tests/ -v
```

## Что задумывалось дальше (после MVP)

- Модель брака и замедления из-за переполнения очередей.
- Предоставление игрокам возможностей более точного планирования путём назначения режимов работы людей и станков (аврал/нормально/работать только когда у следующих по цепочке в очереди много места).
- Моделирование людей и их смен.
- Несколько уровней увеличивающейся сложности.
- Возможности настройки сложности путём установки частоты событий и кол-ва заказов.
- Таблица рекордов.
- Тула-стилизация: пиксельные иконки цеха, шрифт «Iron», копирайт.

## Ограничения MVP

- single-player;
- in-memory state;
- без БД и без авторизации;
- без распределённого/кластерного расчёта;
- упрощённая физика производства (с акцентом на игровой цикл и объяснимость).

## Что внутри

### Описание исходников (`factory/src`)

- `src/main.py`  
  Главная точка входа FastAPI-приложения и REST API для игры: состояние, тик времени, перепланирование, события, копилот, сброс.  
  Здесь же идёт загрузка стартовых данных (`sample_orders.json`) и инициализация in-memory состояния партии.

- `src/models.py`  
  Базовые dataclass-модели домена: станок, операция, заказ, событие, операция в расписании и общее состояние игры.  
  Через `GameState.to_dict()` формируется JSON-ответ API для фронтенда.

- `src/scheduler.py`  
  Планировщик по схеме greedy + EDD: операции раскладываются по станкам с учётом приоритета, дедлайнов и доступности ресурсов.  
  Здесь считается длительность операции с учётом количества (`qty`) и активных событий (перегрузка, повторный ОТК и т.п.).

- `src/events.py`  
  Генератор и обработчик случайных событий из ТЗ (`breakdown`, `supply_delay`, `qc_recheck`, `rush_order`, `no_operator`, `overload`).  
  Отвечает за создание эффектов событий и автоматическое закрытие по времени.

- `src/scoring.py`  
  Логика расчёта 5-компонентного score (`on_time`, `idle`, `balance`, `defects`, `resilience`) и итогового балла.  
  Вызывается после тиков времени, перепланирования и событий, чтобы UI всегда видел актуальную оценку.

- `src/copilot.py`  
  Слой подсказок: обращение к LLM (при наличии ключа) и fallback на заготовленные советы из `src/data/canned_advice.json`.  
  Возвращает краткий совет на русском языке и источник ответа.

### Данные и фронтенд

- `src/data/sample_orders.json`  
  Набор стартовых данных: список станков, рецепты изделий, заказы с дедлайнами/приоритетами/количествами.  
  Этот файл подхватывается при запуске сервера и при `POST /api/reset`.

- `src/data/canned_advice.json`  
  Набор типовых подсказок копилота для офлайн-демо и отказоустойчивости без интернета.  
  Используется как deterministic fallback, чтобы презентационный сценарий не зависел от внешних API.

- `static/index.html`  
  Каркас интерфейса: шапка, toolbar, левая панель станков, правая панель Ганта/score/событий, модальные окна заказов и маршрутов.  
  Подключает `vis-timeline`, внешний CSS и основной фронтенд-скрипт.

- `static/styles.css`  
  Единая таблица стилей интерфейса и тема через CSS-переменные (`--panel-*`, `--bg-*`, `--text`, `--border`).  
  За счёт переменных динамическая смена дня/ночи обновляет все тёмные поверхности одним набором JS-вызовов.

- `static/app.js`  
  Клиентская логика симуляции: опрос API, рендер Ганта и карточек станков, модалки, скорость времени, подсказки копилота.  
  Здесь же реализованы визуальные механики (день/ночь, буферы вход/выход, отображение условных пачек `supplier_*`).

### Тесты и документация

- `tests/test_scheduler.py`  
  Базовые автотесты планировщика и связанных компонентов: корректность расписания, отсутствие overlap, порядок операций, диапазон score, события.  
  Включён регрессионный тест на масштабирование длительности операций по `qty`.

- `docs/factory_spec.md` и `docs/factory_gameplay.md`  
  Живые спецификации проекта: требования кейса, принятые решения, UX/геймплей, roadmap и TODO.  
  По ним удобно сверять, что уже сделано в коде, а что ещё в бэклоге.

## Опциональные этапы (не для MVP)

- **automatic1111**: подключить локальный Stable Diffusion для генерации thumbnail'ов станков/изделий в стиле «industrial steampunk Тула». Заглушка: добавить поле `thumbnail_url` в machines/products и эндпоинт `/api/gen/thumbnail` с обёрткой над `txt2img` API.
- **Уровни сложности**: добавить `state.difficulty`, влияющую на частоту событий и кол-во заказов.

## Главные риски и mitigation

- **Claude API недоступен на стенде** → автоматический fallback на canned-ответы из `canned_advice.json` (3+ заскриптованных ответа на demo-сценарии).
- **Утопиться в Ганте** → vis-timeline через CDN, никакого кастомного SVG.
- **Frontend-полировка съест время** → всё в одном HTML+JS файле, Tailwind CDN не используем.
